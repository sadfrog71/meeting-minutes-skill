# SKILL.md

## Identity
- name: 会议纪要整理
- description: 将原始会议记录整理为结构化、可执行的会议纪要，并支持导出标准格式 Word 文档
- triggers: ["会议纪要", "整理会议", "meeting minutes", "会议记录", "会议总结", "生成会议纪要", "导出会议纪要"]

## Capabilities
1. **会议记录解析** - 从原始记录提取关键信息
2. **结构化整理** - 按标准格式输出会议纪要
3. **Word 文档导出** - 生成专业格式 Word 文档

---

## Instructions

### Phase 1: 输入处理
1. 解析会议基本信息（时间、地点、参会人员、主持人、记录人）
2. 识别会议主题和议题
3. 提取讨论内容和决策事项
4. 识别后续工作安排和责任人

### Phase 2: 结构化整理
按以下格式组织输出：

```
## 基本信息
| 项目 | 内容 |
|------|------|
| 时间 | |
| 地点 | |
| 主持 | |
| 记录 | |

## 会议主题
一句话概括会议目的

## 会议结论
按编号列出所有决策事项，使用**加粗**标注关键人和时间节点

## 责任分工
| 人员 | 任务 |
|------|------|
| | |

## 后续工作
按优先级列出行动项，包含截止时间和负责人
```

### Phase 3: Word 文档生成

当用户要求"导出 Word"、"生成文档"、"下载文档"时，调用 `scripts/generate_docx.py`：

```bash
python scripts/generate_docx.py -o output.docx \
  --company "公司名称" \
  --date "2026年3月17日" \
  --title "公司官网改版讨论会" \
  --time "2026年3月16日 14:00-15:00" \
  --location "2号会议室" \
  --theme "公司官网改版讨论会" \
  --attendees "汪宠、刘朝阳..." \
  --host "汪宠" \
  --absent "马跃" \
  --recorder "刘朝阳"
```

**Word 格式规范：**
- 公司名称 + 日期：居中、黑体、14pt、深蓝
- 会议标题：居中、黑体、16pt、深蓝
- 信息表格：标签列深青底白字，数据列白底宋体
- 章节标题：黑体、12pt、深青，下方留白
- 正文：宋体、10.5pt、1.5倍行距
- 表格：深青表头、白字，边框浅灰

## 输出参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `--company` | 公司名称 | 平行数字 |
| `--date` | 日期 | 2026年3月17日 |
| `--title` | 会议标题 | 公司官网改版讨论会 |
| `--time` | 时间 | 2026年3月16日 14:00-15:00 |
| `--location` | 地点 | 2号会议室 |
| `--theme` | 主题 | 同会议标题 |
| `--attendees` | 参会人员（逗号分隔） | 张三、李四、王五 |
| `--host` | 主持人 | 张三 |
| `--absent` | 请假人员 | 李四 |
| `--recorder` | 记录人 | 王五 |

---

## Constraints
- 只基于提供的原始记录整理，不添加未提及的信息
- 无法确认的信息标注"[待确认]"，不编造
- 涉及人名时保持原始写法
- 时间格式统一为：YYYY年M月D日 H:MM-H:MM
- 附件内容不展开，仅标注"[见附件]"
- Word 导出基于 python-docx，确保输出文件为 .docx 格式
